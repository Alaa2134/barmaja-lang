# كتاب لغة بَرمَجة: دليلك الشامل للبرمجة بالعربية

**المؤلف**: علاء صابر

## المقدمة

في عالم يتزايد فيه الاعتماد على التكنولوجيا والبرمجة، تبرز الحاجة إلى أدوات تعليمية تسهل على المبتدئين دخول هذا المجال. لغة **بَرمَجة** هي مبادرة تهدف إلى سد هذه الفجوة، بتقديم لغة برمجة عربية بالكامل، مبنية على قوة ومرونة JavaScript. هذه اللغة مصممة خصيصًا للمتحدثين باللغة العربية، لتمكينهم من كتابة الكود بلغتهم الأم، مما يزيل حاجز اللغة ويجعل تعلم البرمجة تجربة أكثر سهولة ومتعة.

تخيل أن تكتب `إذا (الطقس === "مشمس") { اطبع("اذهب للنزهة"); }` بدلاً من `if (weather === "sunny") { console.log("Go for a walk"); }`. هذا هو جوهر لغة بَرمَجة: توفير بيئة برمجة طبيعية وبديهية للمبرمج العربي.

يهدف هذا الكتاب إلى أن يكون دليلاً شاملاً للغة بَرمَجة، بدءًا من تصميمها الأساسي، مرورًا بكيفية عمل المترجم الخاص بها، وصولاً إلى استخدام إضافتها في محرر VS Code، وتقديم أمثلة عملية لمساعدتك على البدء.

## الفصل الأول: تصميم لغة بَرمَجة

تعتمد لغة بَرمَجة على استبدال الكلمات المفتاحية الإنجليزية في JavaScript بنظائرها العربية، مع الحفاظ على نفس القواعد النحوية (Syntax) والبنية الأساسية. هذا يضمن أن الكود المكتوب بلغة بَرمَجة يمكن تحويله بسهولة إلى JavaScript قياسي، مما يتيح له العمل في أي بيئة تدعم JavaScript.

### 1.1 الكلمات المفتاحية الأساسية

تم اختيار الكلمات المفتاحية بعناية لتكون واضحة ومعبرة باللغة العربية، مع الحفاظ على أقصر شكل ممكن لتسهيل الكتابة.

| كلمة JavaScript المفتاحية | كلمة بَرمَجة المفتاحية | الوصف |
| :----------------------- | :--------------------- | :---- |
| `var`                    | `متغير`                 | تعريف متغير ذو نطاق وظيفي. |
| `let`                    | `دع`                   | تعريف متغير ذو نطاق كتلي. |
| `const`                  | `ثابت`                  | تعريف ثابت ذو نطاق كتلي. |
| `function`               | `دالة`                  | تعريف دالة. |
| `if`                     | `إذا`                   | تنفيذ كتلة تعليمات برمجية بشكل شرطي. |
| `else`                   | `وإلا`                  | تنفيذ كتلة تعليمات برمجية إذا كان الشرط السابق خاطئًا. |
| `for`                    | `لكل`                   | حلقة تكرارية. |
| `while`                  | `طالما`                 | حلقة تكرارية تستمر طالما كان الشرط صحيحًا. |
| `return`                 | `أرجع`                  | إرجاع قيمة من دالة. |
| `class`                  | `صنف`                   | تعريف صنف (كائن). |
| `this`                   | `هذا`                   | يشير إلى الكائن الحالي. |
| `new`                    | `جديد`                  | إنشاء كائن جديد. |
| `import`                 | `استورد`                | استيراد وحدات (modules). |
| `export`                 | `صدر`                   | تصدير وحدات (modules). |
| `true`                   | `صحيح`                  | القيمة المنطقية صحيح. |
| `false`                  | `خطأ`                   | القيمة المنطقية خطأ. |
| `null`                   | `فارغ`                  | القيمة الخالية. |
| `undefined`              | `غير_معرف`              | القيمة غير المعرفة. |
| `console.log`            | `اطبع`                   | طباعة رسالة إلى وحدة التحكم. |

### 1.2 المعاملات (Operators)

تستخدم لغة بَرمَجة نفس المعاملات القياسية في JavaScript لضمان التوافقية وسهولة التحويل.

| نوع المعامل | المعاملات | الوصف |
| :---------- | :--------- | :---- |
| حسابية      | `+`, `-`, `*`, `/`, `%`, `**` | الجمع، الطرح، الضرب، القسمة، باقي القسمة، الأس. |
| مقارنة      | `==`, `!=`, `===`, `!==`, `>`, `<`, `>=`, `<=` | المساواة، عدم المساواة، المساواة الصارمة، عدم المساواة الصارمة، أكبر من، أصغر من، أكبر من أو يساوي، أصغر من أو يساوي. |
| منطقية      | `&&` (و), `||` (أو), `!` (ليس) | و المنطقية، أو المنطقية، ليس المنطقية. |
| إسناد       | `=`, `+=`, `-=`, `*=`، `/=`, `%=`, `**=` | إسناد قيمة، إسناد مع جمع، إسناد مع طرح، وهكذا. |

### 1.3 بنية الجملة (Syntax Structure)

تتبع بنية الجملة في بَرمَجة بنية JavaScript لتبسيط عملية التحويل:

*   **كتل التعليمات البرمجية**: تُستخدم الأقواس المعقوفة `{}` لتحديد كتل التعليمات البرمجية (مثل كتل `إذا`، `دالة`، `لكل`).
*   **العبارات**: تُنهى كل عبارة بفاصلة منقوطة `;`.
*   **الأقواس**: تُستخدم الأقواس `()` للدوال والشروط.
*   **التعليقات**: 
    *   تعليق سطر واحد: `// هذا تعليق سطر واحد`
    *   تعليق متعدد الأسطر: `/* هذا تعليق
        متعدد الأسطر */`

### 1.4 أنواع البيانات (Data Types)

تدعم بَرمَجة نفس أنواع البيانات الأساسية في JavaScript:

*   **أرقام**: `5`, `3.14` (Numbers)
*   **نصوص**: `"مرحباً بالعالم"`, `"بَرمَجة"` (Strings)
*   **منطقية**: `صحيح`, `خطأ` (Booleans)
*   **مصفوفات**: `[1, 2, 3]` (Arrays)
*   **كائنات**: `{ مفتاح: قيمة }` (Objects)
*   **فارغ**: `فارغ` (Null)
*   **غير_معرف**: `غير_معرف` (Undefined)

## الفصل الثاني: المترجم (Transpiler) الخاص بلغة بَرمَجة

المترجم هو القلب النابض للغة بَرمَجة. وظيفته الأساسية هي تحويل الكود المكتوب بلغة بَرمَجة (التي تستخدم الكلمات المفتاحية العربية) إلى JavaScript قياسي يمكن للمتصفحات أو بيئات التشغيل مثل Node.js فهمه وتنفيذه.

### 2.1 كيف يعمل المترجم؟

يعمل المترجم بخطوات بسيطة:

1.  **قراءة الكود**: يستقبل المترجم الكود المكتوب بلغة بَرمَجة كمدخل.
2.  **تحليل الكلمات المفتاحية**: يمر المترجم على الكود كلمة بكلمة، ويبحث عن الكلمات المفتاحية العربية المحددة في `keywordsMap`.
3.  **الاستبدال**: عندما يجد كلمة مفتاحية عربية، يقوم باستبدالها بنظيرتها الإنجليزية في JavaScript. يتم هذا الاستبدال باستخدام تعبيرات منتظمة (Regular Expressions) لضمان استبدال الكلمات الكاملة فقط وتجنب استبدال أجزاء من كلمات أخرى.
4.  **إخراج الكود**: بعد استبدال جميع الكلمات المفتاحية، يقوم المترجم بإخراج الكود الناتج، والذي يكون الآن JavaScript قياسيًا.

### 2.2 مثال على عملية التحويل

لنأخذ هذا الكود بلغة بَرمَجة:

```barmaja
دالة ترحيب(اسم) {
    إذا (اسم === غير_معرف) {
        أرجع "مرحباً!";
    } وإلا {
        أرجع "مرحباً يا " + اسم + "!";
    }
}
دع رسالة = ترحيب("علاء");
اطبع(رسالة);
```

بعد مرور هذا الكود عبر المترجم، سيتم تحويله إلى JavaScript التالي:

```javascript
function ترحيب(اسم) {
    if (اسم === undefined) {
        return "مرحباً!";
    } else {
        return "مرحباً يا " + اسم + "!";
    }
}
let رسالة = ترحيب("علاء");
console.log(رسالة);
```

كما نرى، تم استبدال `دالة` بـ `function`، و`إذا` بـ `if`، وهكذا. أسماء المتغيرات والدوال التي يحددها المستخدم (مثل `ترحيب` و`اسم`) لا تتغير، مما يسمح للمبرمج بالاحتفاظ بأسماء عربية في الكود المترجم أيضًا.

## الفصل الثالث: إضافة VS Code للغة بَرمَجة

لتوفير تجربة تطوير متكاملة ومريحة، تم تطوير إضافة خاصة بلغة بَرمَجة لمحرر Visual Studio Code. هذه الإضافة توفر العديد من الميزات التي تسهل كتابة الكود بلغة بَرمَجة.

### 3.1 تلوين الصياغة (Syntax Highlighting)

تلوين الصياغة هو ميزة أساسية تساعد على قراءة الكود وفهمه بسرعة. تقوم إضافة بَرمَجة بتلوين الكلمات المفتاحية، النصوص، الأرقام، والتعليقات بألوان مختلفة، مما يجعل الكود أكثر وضوحًا وجاذبية.

### 3.2 الإكمال التلقائي (IntelliSense)

توفر الإضافة ميزة الإكمال التلقائي للكلمات المفتاحية العربية. عند البدء في كتابة كلمة مفتاحية، ستقترح الإضافة الكلمة الكاملة، بالإضافة إلى عرض نظيرتها في JavaScript ووصف موجز لها. هذا يسرع عملية الكتابة ويقلل من الأخطاء.

### 3.3 أوامر التحويل والتشغيل

تتضمن الإضافة أمرين رئيسيين يمكن الوصول إليهما من لوحة الأوامر (Command Palette) أو عبر اختصارات لوحة المفاتيح:

*   **`Barmaja: Transpile to JavaScript` (اختصار: `Ctrl+Shift+T` أو `Cmd+Shift+T` على Mac)**:
    يقوم هذا الأمر بتحويل الكود المكتوب في الملف الحالي بلغة بَرمَجة إلى JavaScript قياسي، ويحفظه في ملف جديد بنفس اسم الملف الأصلي ولكن بامتداد `.js`.

*   **`Barmaja: Run Code`**:
    يقوم هذا الأمر بتحويل الكود الحالي إلى JavaScript بشكل مؤقت، ثم يقوم بتشغيله باستخدام Node.js في طرفية VS Code المدمجة. هذا يسمح لك باختبار الكود الخاص بك بسرعة ورؤية النتائج مباشرة.

### 3.4 كيفية تثبيت الإضافة (بعد النشر)

بمجرد نشر الإضافة على VS Code Marketplace، يمكنك تثبيتها مباشرة من داخل VS Code:

1.  افتح VS Code.
2.  انتقل إلى قسم الإضافات (Extensions) من الشريط الجانبي (Ctrl+Shift+X).
3.  ابحث عن 
`لغة بَرمَجة` أو `Barmaja Language`.
4.  انقر على زر `تثبيت` (Install).

## الفصل الرابع: أمثلة عملية وتطبيقات

في هذا الفصل، سنستعرض مجموعة من الأمثلة العملية التي توضح كيفية استخدام لغة بَرمَجة في سيناريوهات مختلفة، بدءًا من الأساسيات وصولاً إلى مفاهيم أكثر تقدمًا.

### 4.1 مثال بسيط: برنامج الترحيب

```barmaja
// مثال بسيط: برنامج الترحيب
// Hello World Program in Barmaja

دالة ترحيب(اسم) {
    إذا (اسم === غير_معرف || اسم === فارغ) {
        أرجع 'مرحباً بك في لغة بَرمَجة!';
    } وإلا {
        أرجع 'مرحباً يا ' + اسم + '! أهلاً وسهلاً في بَرمَجة.';
    }
}

// استدعاء الدالة
اطبع(ترحيب('علاء صابر'));
اطبع(ترحيب());

// مثال على المتغيرات والثوابت
دع اليوم = 'الجمعة';
ثابت السنة = 2026;

اطبع('اليوم: ' + اليوم);
اطبع('السنة: ' + السنة);
```

**الشرح**:

*   نبدأ بتعريف `دالة` باسم `ترحيب` تستقبل `اسم` كمعامل.
*   داخل الدالة، نستخدم `إذا` و`وإلا` للتحقق مما إذا كان `اسم` `غير_معرف` أو `فارغ`.
*   نستخدم `أرجع` لإعادة قيمة من الدالة.
*   نستخدم `دع` لتعريف متغير يمكن إعادة تعيين قيمته، و`ثابت` لتعريف ثابت لا يمكن تغيير قيمته بعد التعريف.
*   `اطبع` هي اختصار لـ `console.log` وتستخدم لطباعة المخرجات إلى وحدة التحكم.

### 4.2 مثال متقدم: الفئات (Classes) والحلقات (Loops)

```barmaja
// مثال متقدم: فئات وحلقات
// Advanced Example: Classes and Loops

صنف طالب {
    جديد(اسم, درجة) {
        هذا.اسم = اسم;
        هذا.درجة = درجة;
    }

    دالة احصل_على_التقدير() {
        إذا (هذا.درجة >= 90) {
            أرجع 'ممتاز';
        } وإلا إذا (هذا.درجة >= 80) {
            أرجع 'جيد جداً';
        } وإلا إذا (هذا.درجة >= 70) {
            أرجع 'جيد';
        } وإلا {
            أرجع 'ضعيف';
        }
    }

    دالة اطبع_البيانات() {
        اطبع('الاسم: ' + هذا.اسم);
        اطبع('الدرجة: ' + هذا.درجة);
        اطبع('التقدير: ' + هذا.احصل_على_التقدير());
    }
}

// إنشاء قائمة من الطلاب
دع الطلاب = [
    جديد طالب('علاء', 95),
    جديد طالب('فاطمة', 88),
    جديد طالب('محمد', 76),
    جديد طالب('نور', 92)
];

// طباعة بيانات كل طالب
لكل (دع i = 0; i < الطلاب.length; i++) {
    اطبع('--- الطالب ' + (i + 1) + ' ---');
    الطلاب[i].اطبع_البيانات();
    اطبع('');
}

// حساب المتوسط
دع مجموع_الدرجات = 0;
لكل (دع i = 0; i < الطلاب.length; i++) {
    مجموع_الدرجات = مجموع_الدرجات + الطلاب[i].درجة;
}

دع المتوسط = مجموع_الدرجات / الطلاب.length;
اطبع('متوسط الدرجات: ' + المتوسط.toFixed(2));
```

**الشرح**:

*   نستخدم `صنف` لتعريف فئة `طالب` تحتوي على خصائص (مثل `اسم` و`درجة`) ودوال (مثل `احصل_على_التقدير` و`اطبع_البيانات`).
*   `جديد` تستخدم لإنشاء كائنات جديدة من الفئة.
*   `هذا` تشير إلى الكائن الحالي داخل الفئة.
*   نستخدم حلقة `لكل` للتكرار على مصفوفة `الطلاب` وطباعة بيانات كل طالب وحساب متوسط الدرجات.

## الفصل الخامس: المساهمة والتطوير المستقبلي

لغة بَرمَجة هي مشروع مفتوح المصدر (Open Source) يرحب بمساهمات المجتمع. إذا كنت مهتمًا بالمساعدة في تطوير اللغة أو الإضافة، فهناك العديد من الطرق التي يمكنك من خلالها المساهمة.

### 5.1 كيفية المساهمة

1.  **الإبلاغ عن الأخطاء (Bug Reports)**: إذا وجدت أي أخطاء في المترجم أو الإضافة، يرجى الإبلاغ عنها عبر صفحة المشروع على GitHub.
2.  **اقتراح الميزات (Feature Suggestions)**: لديك فكرة لتحسين اللغة أو الإضافة؟ شاركها معنا!
3.  **كتابة الكود (Code Contributions)**: يمكنك المساهمة في كتابة الكود للمترجم، الإضافة، أو حتى الأمثلة والتوثيق.
4.  **تحسين التوثيق**: المساعدة في تحسين هذا الكتاب أو أي توثيق آخر للغة.

### 5.2 خارطة الطريق المستقبلية

نطمح لتطوير لغة بَرمَجة لتصبح أداة قوية ومتكاملة. تتضمن خططنا المستقبلية:

*   **دعم المزيد من الكلمات المفتاحية**: إضافة المزيد من الكلمات المفتاحية العربية لتغطية جوانب أوسع من JavaScript.
*   **تحسينات الأداء**: العمل على جعل عملية التحويل أسرع وأكثر كفاءة.
*   **أدوات تصحيح الأخطاء (Debugging Tools)**: تطوير أدوات لمساعدة المبرمجين على اكتشاف الأخطاء وإصلاحها بسهولة.
*   **نظام إدارة الحزم (Package Manager)**: إنشاء نظام لإدارة المكتبات والوحدات الخاصة بلغة بَرمَجة.
*   **دعم لغات إضافية**: التوسع لدعم لغات أخرى غير العربية، مما يجعل المشروع متعدد اللغات.
